
# 📄 配置文件格式说明 (JSON)

该 JSON 文件由两个主要部分组成：`config`（全局配置）和 `content`（对话内容列表）。

---

## 1. 整体结构

```json
{
  "config": { ... },
  "content": [ ... ]
}

```

---

## 2. 字段详细说明

### A. `config` 对象 (可选)

用于定义页面的基础信息。

* **`title`** (String): 页面标题。会显示在浏览器的标签页以及页面的主标题位置。
* **`subtitle`** (String): 页面副标题。通常显示在标题下方，如“2人在线”。

### B. `content` 数组 (必填)

包含所有要显示的消息或组件。数组中的每个元素是一个对象，根据 `type` 或 `role` 区分显示样式。

**消息对象属性：**

* **`role`** (String): 发言者角色。
* `"清言"`: 触发左侧/特定头像及样式。
* `"熊"`: 触发右侧/特定头像及样式。
* *注意：代码通过判断角色名来分配 CSS 类名。*

* **`text`** (String): 消息的正文内容。
* 支持简单的 Markdown 格式：使用双星号（如 `**醒觉静息效应**`）会被解析为加粗文本（`<b>` 标签）。

* **`type`** (String, 可选): 元素类型。
* 如果值为 `"divider"`，则该项被视为**分割线**。此时 `role` 属性无效，页面将居中显示 `text` 内容。

---

## 3. 示例代码

以下是一个包含所有可能情况的精简示例：

```json
{
  "config": {
    "title": "清言与熊的备考秘籍",
    "subtitle": "2人在线"
  },
  "content": [
    {
      "role": "熊",
      "text": "你好，清言！"
    },
    {
      "type": "divider",
      "text": "以下是科普内容"
    },
    {
      "role": "清言",
      "text": "你好！你知道 **醒觉静息** 吗？"
    }
  ]
}

```
